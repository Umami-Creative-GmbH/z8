# Tolgee Translations (Namespaced)

Translations use namespace-based code splitting.

## Namespace Structure

| Namespace | Key Prefixes | Routes |
|-----------|-------------|--------|
| `common` | common, generic, nav, header, user, table, validation, errors, info, meta | All pages |
| `auth` | auth, profile, sessions | /sign-in, /sign-up |
| `dashboard` | dashboard | / |
| `calendar` | absences, calendar | /calendar, /absences |
| `timeTracking` | timeTracking, wellness | /time-tracking |
| `reports` | reports | /reports |
| `settings` | settings, organization, vacation, team | /settings/** |
| `onboarding` | onboarding | /onboarding/** |

## File Structure

```
messages/
├── common/en.json, de.json, ...     # ~5 KB - always loaded
├── auth/en.json, de.json, ...       # ~8 KB
├── settings/en.json, de.json, ...   # ~52 KB - only on /settings
└── ...
```

## Adding Keys

Keys auto-assign to namespaces by prefix:
- `settings.employees.title` → `settings`
- `dashboard.widgets.vacation` → `dashboard`
- `common.cancel` → `common`

## Loading Namespaces Client-Side

```typescript
import { useNamespaces } from "@/tolgee/client";

function MyComponent() {
  const { isLoading } = useNamespaces(["settings", "calendar"]);
  if (isLoading) return <Spinner />;
  return <div>{t("settings.title")}</div>;
}
```

## Key Files

- `tolgee.config.cjs` - CLI config
- `src/tolgee/shared.ts` - Namespace loaders and route mappings
- `src/tolgee/client.tsx` - Provider and `useNamespaces` hook
- `tolgee-extractor.mjs` - Custom extractor with namespace inference

## CLI Commands

```bash
pnpm dlx @tolgee/cli extract print  # Extract keys
pnpm dlx @tolgee/cli sync           # Sync with server
pnpm dlx @tolgee/cli pull           # Pull translations
pnpm dlx @tolgee/cli push           # Push translations
```
