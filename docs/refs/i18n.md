# i18n Reference

## Open This When

- You add or change translation keys.
- You update namespace routing or message loading.
- You need to debug extractor behavior.

## Read First

- `apps/webapp/src/tolgee/shared.ts`: namespace model and route-based namespace loading.
- `apps/webapp/src/tolgee/server.tsx`: server translation entrypoints.
- `apps/webapp/src/tolgee/client.tsx`: client provider and namespace hydration.
- `apps/webapp/tolgee.config.cjs`: Tolgee CLI config.
- `apps/webapp/tolgee-extractor.mjs`: custom key extraction and namespace inference.
- `apps/webapp/messages/`: namespace-based locale catalogs.

## Key Rules

1. Keep keys stable and statically extractable.
2. Use namespace-prefixed keys when needed.
3. Provide fallback defaults in `t()` calls.
4. Keep extractor namespace mapping aligned with `messages/<namespace>/<locale>.json`.

## Workflow

1. Add or update keys in code.
2. Ensure namespace mapping is correct.
3. Sync/update message files through the existing Tolgee workflow.
